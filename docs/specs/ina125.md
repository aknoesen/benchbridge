# Spec — INA125 instrumentation amplifier (the only in-amp)

Decision (andre, 2026-06-27): the **only** instrumentation amplifier in the app is the real **INA125**
(no package-less ideal/3-op-amp in-amp). It is modeled **structurally** so the student's external gain
resistor R_G sets the gain. Mirrors the op-amp → LMC662 decision.

Datasheet: TI/Burr-Brown INA125 (SBOS060 / PDS-1361B). Verified from `ti.com/lit/ds/symlink/ina125.pdf`.

## Verified facts

- 16-pin PDIP (INA125P). Single supply +2.7..+36 V or dual ±1.35..±18 V.
- **Gain = 4 + 60 kΩ / R_G**, R_G external across **pins 8 and 9**. R_G open ⇒ G = 4 (min).
- Output referred to **IAREF** (pin 5), normally grounded.
- On-chip precision reference: connect **VREFOUT (pin 4)** to one of **VREFBG (1.24 V, pin 13)**,
  **VREF2.5 (pin 14)**, **VREF5 (pin 15)**, **VREF10 (pin 16)**; common is **VREFCOM (pin 12)**.
- VO = pin 10, Sense = pin 11 (Kelvin; tie to VO for normal use). SLEEP = pin 2 (tie to V+ if unused).

### Pinout (pin 1 at notch, top view)

| 1 V+ | 2 SLEEP | 3 V− | 4 VREFOUT | 5 IAREF | 6 VIN− | 7 VIN+ | 8 RG |
|------|---------|------|-----------|---------|--------|--------|------|
| 16 VREF10 | 15 VREF5 | 14 VREF2.5 | 13 VREFBG | 12 VREFCOM | 11 Sense | 10 VO | 9 RG |

## Simulation — structural model (matches the gain equation via external R_G)

A 3-op-amp in-amp whose resistor values reproduce **G = 4 + 60 kΩ/R_G** with the *external* R_G:
- First stage: buffers A1 (VIN+), A2 (VIN−); feedback **7.5 kΩ** from each buffer output to its R_G
  node (pins 8/9); external R_G bridges the two R_G nodes. First-stage diff gain = 1 + 2·7.5k/R_G.
- Difference amp (A3): **R2 = 10 kΩ, R3 = 40 kΩ** → stage gain 4, referred to IAREF (pin 5), Sense=VO.
- Net: (1 + 15k/R_G)·4 = **4 + 60 kΩ/R_G**. ✓ (R_G open ⇒ 4.)
- Each internal op-amp = the existing behavioral op-amp macro (finite gain/BW, clip to V+/V−).
- Reference: a voltage source on VREFOUT vs VREFCOM whose value = the VREF pin strapped to VREFOUT
  (1.24/2.5/5/10 V); if none strapped, model the bandgap and warn (datasheet says always strap one).
- Power V+/V− implied in sim (auto ±15 V, or PSU rails); exposed as DIP pins on the board.

Add `ina125Lines(c,n)` in `netlist.ts`; emit an `ina125` circuit component from `toCircuit`.

## Breadboard

16-pin DIP = **8 columns** (extend `dipCols`/`dipPinHoles`). Pinout legend in the side panel like the
LMC662 one (full 16-pin map, V+/V− colored). Check: map the schematic INA125's used signal pins
(VIN−/VIN+/RG/RG/VO) to the DIP pins; require V+/V− on the rails. External R_G is a normal placed
resistor across pins 8/9.

### Auxiliary-pin straps (2026-06-27, from Lab 8 Fig 1 / datasheet)

The schematic symbol stays abstract (the functional pins), but the **board Check enforces the
datasheet-mandated chip wiring** the real INA125 needs to function — so students wire every pin
(Lab 8 Prelab Deliverable #5). These are fixed `straps` on the DIP expectation, checked like the
rails, independent of the schematic:

| Pin | Strap to | Why |
|-----|----------|-----|
| SLEEP (2) | V+ rail | else the device stays in shutdown |
| VREFout (4) | VREF2.5 (14) | strap the on-chip reference (else the reference amp saturates) |
| IAref (5) | GND | sets the output reference |
| Sense (11) | Vo (10) | close the output-buffer feedback (Kelvin) |
| VREFcom (12) | GND | reference common |

Plus V+ (1) / V− (3) on the supply rails. IAref is enforced via the strap (not the schematic), so
the abstract symbol need not expose it. The pinout legend lists the required strapping; the Check
returns a per-strap hint (e.g. "Wire U1: Sense (pin 11) → Vo (pin 10) …"). Covered by tests in
`breadboard.test.ts` (full-strap pass + missing-strap flags).

## Remove

Drop `inamp` / `inamp3` kinds and their toolbar/picker/netlist paths (package-less). INA125 replaces them.

## Course usage (confirmed with andre, 2026-06-27)

- **Dual supply** (from the M2K V+/V− rails, ±5 V), **IAREF = GND** (output referred to ground).
- **Moderate gain, G ≈ 10–100** (R_G ≈ 10 kΩ down to ~625 Ω). On-chip reference used only when a
  circuit needs it (not the default).

Worked example: a dual-supply differential amplifier at **G ≈ 10** (R_G = 10 kΩ), IAREF grounded,
small differential drive so the output stays within ±5 V. Show CH2 = differential input, CH1 = output.

## Why not the TI PSpice macromodel (decided 2026-06-27)

Settled: keep the **structural** model; do **not** import TI's PSpice subcircuit. Reasons: PSpice
syntax may not run unmodified in this WASM ngspice build; the vendor model adds offset/CMRR/noise
that fights the "ideal twin" pedagogy and would confuse first-years; it is heavier (convergence +
the ~10 Hz re-sim loop); the 3-op-amp structure is transparent/teachable where a macromodel is a
black box; and TI model files carry redistribution terms awkward for a public repo. If real
accuracy is ever needed, run a proper SPICE simulator instead of the twin. (Possible future: load a
TI `.subckt` as an *optional* advanced part, not the default.)

## Model validation (2026-06-27)

Bench-checked the structural topology in ngspice (ideal op-amps): R_G open → G = 4.00; 10 kΩ → 10.00;
1.3 kΩ → 50.15; 625 Ω → 100.00 — exact match to 4 + 60 kΩ/R_G. Topology + resistor values locked.

## Acceptance

- Place INA125 + external R_G; sim shows VO = (4 + 60k/R_G)(VIN+ − VIN−) + V(IAREF), clipped to rails.
- Changing R_G changes gain per the equation. Boards as a 16-pin DIP; Check enforces V+/V− on rails.
- tsc clean; 12-bit canary untouched (no core/signal.ts).
